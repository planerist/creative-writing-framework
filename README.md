# Novel-Framework (Antigravity)

Среда на основе базы знаний для написания и сопровождения сериального романа глава за главой с помощью AI-агентов, построенная под **[Google Antigravity](https://antigravity.google)**.

Движок письма — это набор **workflows** (slash-команд) плюс **rules**, поверх отдельной для каждого романа **базы знаний**. Готовый, **жанрово-нейтральный демо-роман** лежит в `novels/_template/`, чтобы вы могли увидеть всю машину целиком, прежде чем писать свою.

> **Заточено под Antigravity.** Правила живут в `AGENTS.md` + `.agents/rules/`; движок живёт в `.agents/workflows/`. 

---

## Структура

```
.
├── AGENTS.md                     # постоянные правила взаимодействия (Antigravity их читает)
├── .agents/
│   ├── rules/
│   │   └── writing-craft.md      # Always-On ограничения по ремеслу (без длинного тире, show-don't-tell, …)
│   └── workflows/                # движок письма, вызывается как /slash-команды
│       ├── draft.md      refine.md    analyze.md    beta-read.md   challenge.md
│       └── snapshot.md   meta.md      sync-kb.md    fix-lang.md
├── craft/
│   ├── prose_failure_modes.md    # универсальный слой провалов AI-прозы (читается при написании/анализе)
│   └── analysis_passes.md        # методология разбора главы (читается /analyze)
└── novels/
    └── _template/                # жанрово-нейтральное демо — скопируйте, чтобы начать настоящий роман
        ├── premise.md
        ├── knowledge/            # база знаний (канон) — сначала прочтите knowledge/index.md
        └── chapters/             # chapter_1 проработана целиком + заготовка плана chapter_2
```

Сгенерированная проза, краткие описания, снимки состояния и отчёты всегда пишутся на **русском** языке.

## Workflows (движок письма)

Вызывайте каждый в Agent slash-командой. Workflows могут вызывать друг друга.

| Workflow     | Что делает |
| ------------ | --- |
| `/challenge` | Испытывает новую идею на прочность против всей базы знаний, прежде чем вы её зафиксируете (критика из 5 пунктов). Также срабатывает автоматически по Always-On правилу ремесла. |
| `/draft`     | Пишет черновик главы по её плану, удерживая состояние персонажей, прошлые главы, базу знаний и слой ремесла. |
| `/refine`    | Извлекает встроенные авторские пометки (`[[ ]]`, `` ` ` ``) и применяет их. |
| `/snapshot`  | Создаёт файл состояния на конец главы, из которого читает следующая глава. |
| `/meta`      | Пишет краткое описание главы и подбирает теги. **См. оговорки.** |
| `/sync-kb`   | Предлагает изменения ADD/UPDATE/REMOVE для базы знаний после новой главы, в виде готового к копированию отчёта, который вы затем просматриваете и применяете. |
| `/analyze`   | Полный разбор готовой главы по многим аспектам (логика мира, стиль, темп, непрерывность, повторы и др.); сохраняет отчёт рядом с главой. |
| `/beta-read` | Симулирует поминутный опыт читателя; дополняет аналитический разбор. |
| `/fix-lang`  | Цикл «сканируй, затем правь» для механических языковых проблем (длинные тире, лимит абзацев, повторы), до 5 проходов. |

## Как создаётся глава

1. Напишите `chapters/chapter_N_outline.md` (биты списком; попросите агента помочь).
2. Запустите **`/draft`**: он читает `chapter_{N-1}_snapshot.md`, план, прошлые главы, всю `knowledge/` и `craft/prose_failure_modes.md`, затем пишет черновик `chapter_N.md`.
3. Проверьте через **`/analyze`** и/или **`/beta-read`**; оставьте встроенные пометки и запустите **`/refine`**; приведите в порядок механику через **`/fix-lang`**.
4. Запустите **`/snapshot`** и **`/meta`**, чтобы создать `chapter_N_snapshot.md` и `chapter_N_meta.md`.
5. Запустите **`/sync-kb`**, чтобы получить отчёт с предлагаемыми изменениями, затем просмотрите и примените его, чтобы влить новый канон в базу знаний.

В `novels/_template/chapters/` глава 1 проработана целиком (план, черновик, снимок состояния, мета), а для главы 2 готов только план — заготовка для следующего цикла.

## Как начать свой роман

```
cp -R novels/_template novels/<ваш-роман>
```

Сохраните структуру, замените содержимое: перепишите `premise.md` и каждый файл в `knowledge/` (особенно лист стиля и `writing_axes.md`). Агент выводит роман из контекста или спрашивает; вся работа остаётся внутри `novels/<ваш-роман>/`.

## Оговорки / что стоит локализовать

- **`/meta` поставляется с примером таксономии тегов** (небольшой литературный список). Замените его словарём тегов вашей площадки или держите его в `knowledge/tags.md` и укажите на него в workflow.
- **Движок по умолчанию жанрово-нейтрален.** `/analyze` и `/beta-read` оценивают главу по тем критериям, которые ВЫ задаёте в базе знаний своего романа (файл `writing_axes` и описанный там читательский контракт). Конкретный жанр (триллер, романтика, эротика…) настраивает движок, добавляя свои критерии в свою базу знаний, а не редактируя workflows.
- **Весь текст и методология — на русском.** На английском оставлены только технические термины Antigravity (workflow, rules, AGENTS.md, имена slash-команд) и имена файлов — такими вы видите их в IDE. Язык вывода задан в `.agents/rules/writing-craft.md` (по умолчанию русский).
- **Antigravity ограничивает каждый файл rule/workflow примерно 12 000 символами.** Объёмная методология анализа живёт в `craft/` и подключается тонким workflow, чтобы оставаться под лимитом. Сохраняйте этот приём, если расширяете workflow.
