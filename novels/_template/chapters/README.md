# `chapters/` — рукопись и её артефакты

Эта папка хранит прозу и файлы, которые агенты производят вокруг неё для каждой главы.
Каждая глава `N` — это небольшой набор файлов, работающих вместе, так чтобы *следующую*
главу можно было написать с полной непрерывностью.

## Файлы для каждой главы

| Файл                    | Что это                                                                                                                                       | Кем производится |
| ----------------------- | --------------------------------------------------------------------------------------------------------------------------------------------- | --- |
| `chapter_N_outline.md`  | План главы автора по пунктам (биты). Вход для отрисовки.                                                                                      | Вами (с помощью агента). |
| `chapter_N.md`          | Собственно проза главы.                                                                                                                       | `/draft`, из плана главы. |
| `chapter_N_snapshot.md` | Физическое / эмоциональное / социальное состояние каждого персонажа на **конец** главы, плюс флаги непрерывности. Следующая глава читает это. | `/snapshot`. |
| `chapter_N_meta.md`     | Короткое резюме и теги главы.                                                                                                                 | `/meta`. |
| `chapter_N_*.tmp.md`    | Эфемерные рабочие файлы (извлечённые авторские пометки, предложения по обновлению KB). Пересоздаются при каждом запуске; в git не входят.     | Разные workflow, проходяще. |

## Цикл

```
1. план главы        →  chapter_N_outline.md   (пишете вы)
2. /draft            →  chapter_N.md           (черновик по плану)
3. /analyze, /beta-read                         (разбор; правки через /refine, /fix-lang)
4. /snapshot         →  chapter_N_snapshot.md  (состояние на конец главы)
   /meta             →  chapter_N_meta.md      (описание и теги)
5. /sync-kb                                     (обновить базу знаний)

   → chapter_N_snapshot.md читает следующая глава, когда /draft запускается снова
```

После разбора вы можете оставить авторские пометки (`[[ ]]`, `` ` ` ``) в `chapter_N.md` и
запустить `/refine`, чтобы их применить, или запустить `/fix-lang` для прохода
механической чистки.

## Что есть в этом демо

- **Глава 1 проработана от начала до конца:** `chapter_1_outline.md`, `chapter_1.md` (полная
  проза), `chapter_1_snapshot.md`, `chapter_1_meta.md`.
- **Глава 2 — заготовка плана** (`chapter_2_outline.md`) — отправная точка следующего
  цикла, показывающая, как состояние переносится вперёд из снимка состояния главы 1. Прозы
  главы 2 в демо нет.
