# `knowledge/` — база знаний (KB)

**Канон** романа: незыблемые факты о мире, персонажах, правилах и стиле. Skills читают
**все** эти файлы, прежде чем писать или анализировать главу, поэтому база знаний — это
способ держать преемственность, не удерживая всё в голове.

**Начинайте с [`index.md`](index.md)**: это оглавление, которое сообщает агенту, что
держит каждый файл и когда к нему обращаться.

## Как это устроено

- **Нумерованные файлы, нестрогое соглашение.** Файлы снабжены префиксами `00_`,
  `01_`, … чтобы читаться в разумном порядке. Цифры не волшебные; именно `index.md`
  их сопоставляет.
- **Канон, а не проза.** Сюда идут факты и правила, а не сцены. Когда глава
  устанавливает новый факт (имя, место, изменившиеся отношения), верните его обратно
  через `/sync-kb` (команда предлагает изменения, которые вы просматриваете и
  применяете). **Никогда** не переписывайте прошлые главы под изменившийся факт;
  история движется вперёд, а база знаний фиксирует изменение.
- **Два файла выполняют особую работу**, и большинство workflow опираются на них
  по имени или по роли:
  - файл **стиля и языка** (здесь `03_language_and_style.md`) — неоспоримый свод
    правил голоса, проверяемый в каждом сеансе письма и редактуры;
  - файл **осей письма** (здесь `04_writing_axes.md`) — контрольный список качества,
    по которому каждый разбор оценивает сцену.

## Файлы в этой демонстрационной базе знаний

| Файл                          | Что держит |
| ----------------------------- | --- |
| `index.md`                    | Оглавление базы знаний. Читать первым. |
| `00_setting.md`               | Мир, места, центральное давление, факты о времени и пути, условности. |
| `01_characters_main.md`       | Полные профили главных героев: внешность, психология, история, желания против страхов, телесные эпитеты. |
| `02_characters_background.md` | Второстепенный состав, намеренно лёгкий. |
| `03_language_and_style.md`    | Свод стиля: точка зрения (POV), время, голос, диалог, запрещённые тики, правило «никаких длинных тире». |
| `04_writing_axes.md`          | Оси качества, по которым диагностируется каждая сцена. |
| `05_vocabulary.md`            | Банк конкретной фактуры: конкретные существительные и глаголы, к которым стоит тянуться, и клише, которых надо избегать. |

База знаний вашего настоящего романа будет иметь собственные файлы. Держите как минимум
`index.md`, файл стиля и файл осей письма; добавляйте столько файлов о мире, персонажах
и правилах, сколько нужно истории.
