# Fix Language

Только механическая чистка языка, НЕ суждение о стиле/голосе (это живёт в
`/analyze`). Сканируйте проблемы Прохода 6 (Язык), применяйте их и повторяйте, пока
глава не стабилизируется.

> Скан и правка происходят встроенно в одном workflow. Держите две фазы мысленно
> раздельно: полностью перечислите замечания ДО редактирования, чтобы правки не сдвигали
> текст из-под более поздних замечаний в том же проходе.

## Шаги

1. Определите путь к главе из вызова (спросите, если он не задан). Используйте абсолютный путь.

2. **Цикл, до 5 проходов.** Каждый проход:

   **(a) СКАН — ТОЛЬКО Проход 6 (Язык).** Не запускайте другие проходы. Читайте только файл главы и KB романа по стилю/языку (файлы style и vocabulary в `knowledge/`). Прежде чем объявить категорию чистой, ПОСЧИТАЙТЕ сырые кандидаты (используйте `grep` для длинных тире, повторов оборотов и т.п.), никогда не определяйте на глаз. Подтвердите каждый кандидат по прозе. Соберите замечания в список; для каждого запишите:
   - **LOCATION** — дословный фрагмент из главы, уникальный в файле (включите ~3 слова контекста с каждой стороны). Для разбивки абзаца используйте короткий фрагмент, оседлавший шов (конец предложения 1 + начало предложения 2), а не весь абзац.
   - **DESCRIPTION** — короткая метка ("длинное тире как разделитель действия", "абзац превышает лимит ~150 слов", "шаблон коллокации заезжен", "обращение не с заглавной буквы в прямом обращении", "повтор глагола: руки + дрожали (3-й раз)").
   - **FIX** — ровно одно из: `REPLACE WITH: <текст>` / `DELETE` / `SPLIT HERE` / `CAPITALIZE: <слово>`.

   Что охватывает Проход 6 (по `craft/analysis_passes.md`): длинные тире (только символ `—` U+2014, запрещённый в прозе этого framework, если только KB стиля романа не разрешает ограниченное использование), лимит длины абзаца, повторяющиеся шаблоны коллокаций (≥3×), монотонность порядка осмотра тела, повторы `[часть тела] + [глагол]` (≥3×), регистр обращений и конвенцию оформления реплик.

   **(b) Если ни одно замечание не заслуживает флага → STOP.** Запишите `pass N: 0 fixes (complete)` и перейдите к шагу 3.

   **(c) ПРАВКА.** Примените каждое замечание как одно редактирование на его LOCATION:
   - `REPLACE WITH: <текст>` → замените LOCATION на `<текст>`.
   - `DELETE` → удалите LOCATION.
   - `SPLIT HERE` → вставьте разрыв абзаца (`\n\n`) на границе предложений внутри LOCATION.
   - `CAPITALIZE: <слово>` → сделайте это слово с заглавной буквы внутри LOCATION.
   Если LOCATION не уникален (совпадает в нескольких местах), ПРОПУСТИТЕ его (засчитайте как пропущенный), не угадывайте и не расширяйте совпадение. Сохраняйте Markdown-маркеры (`*курсив*`, заголовки, списки), если только правка не требует их изменения. Запишите счётчики `applied` и `skipped` и описания.

   **(d) STOP** при любом из: ноль применённых, все замечания пропущены, два последовательных прохода с одинаковыми счётчиками по категориям (неподвижная точка) или достигнут проход 5. Иначе вернитесь к (a).

3. **Напечатайте сводку** (≤15 строк; не повторяйте полный список замечаний):
   ```
   fix-lang — <путь к главе>
   проходов выполнено: <N> / 5
   всего правок: <сумма>

   проход 1: applied=<n> skipped=<n> — <напр., "2 разбивки абзацев, 3 длинных тире">
   проход 2: ...

   итоговый статус: <complete | iteration cap | fixed-point>
   ```

## НЕ делайте
- НЕ выносите суждений о стиле/голосе: дрейф регистра/голоса относится к `/analyze`.
- НЕ зацикливайтесь дольше 5 проходов.
- НЕ редактируйте LOCATION, который не совпадает однозначно: пропустите и сообщите о нём.
